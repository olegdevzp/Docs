# Indie Solo Mobile App Promotion Roadmap (From 0)

> A practical, bootstrapped path for one person — no agency, no big ad budget.

Assumes a **utility/education app** (e.g. first aid reference): trust and discoverability matter more than viral hype.

---

## Phase 0: Foundation (Before You Promote Anything)

**Goal:** Be promotable when launch day comes.

| Task | Why |
|------|-----|
| Pick one primary platform first (iOS *or* Android) | Split focus kills solo founders |
| Define one sentence: *"This app helps [who] do [what] better than [alternative]"* | Every post, store listing, and pitch uses this |
| Set up: landing page, email waitlist (Buttondown/ConvertKit free tier), 1 social account | You need somewhere to send people |
| Prepare store assets: icon, 5–8 screenshots, 30-sec screen recording | ASO and shares depend on this |
| Install analytics: Firebase + App Store Connect / Play Console | You can't improve what you don't measure |
| Build in public doc: problem → solution → progress updates | Content for later phases |

**Budget:** $0–$200 (domain + optional email tool)

**Time:** 2–4 weeks alongside development

---

## Phase 1: Pre-Launch (8–12 Weeks Before Release)

**Goal:** 100–500 waitlist signups + store listing ready.

### Week 1–4: ASO Prep (Free, High ROI)

```
Keyword research → Title + subtitle → Description → Screenshots with benefit text
```

- Use free tools: AppFollow, AppTweak trial, or manual App Store search
- Target **long-tail keywords** ("first aid bleeding guide", not "health app")
- Write description for humans first, keywords second
- Localize at least **English + your main market language** if relevant

### Week 4–8: Content Seeding

| Channel | Solo-Friendly Tactic |
|---------|----------------------|
| **Twitter/X or LinkedIn** | 3 posts/week: problem tips, build updates, "why I'm building this" |
| **Reddit** | Answer questions in r/firstaid, r/Preppers, r/survival — *no spam* |
| **Medium / Dev.to / personal blog** | 2 articles: "What I learned building X" + "How to handle Y" (real value) |
| **YouTube Shorts / TikTok** | 15-sec tips from your app content (repurpose, don't sell) |

**Rule:** 80% value, 20% "I'm building an app for this."

### Week 8–12: Soft Launch Prep

- [ ] TestFlight / closed beta with 20–50 people (friends, Reddit volunteers, Facebook groups)
- [ ] Fix crash reports and confusing UX — bad reviews at launch are hard to undo
- [ ] Ask beta users for **2-sentence testimonials** and App Store review on day 1
- [ ] Schedule launch posts (don't improvise launch day)

**Budget:** $0–$500 (optional: Canva Pro, ASO tool for 1 month)

**KPIs:** Waitlist size, beta feedback score, keyword rankings in sandbox

## Phase 2: Launch Week (Days 1–7)

**Goal:** First 200–1,000 installs + 10–30 reviews.

### Day 0 Checklist

```
Store live → Email waitlist → Product Hunt → Reddit (where allowed) → Personal network → Press micro-pitch
```

| Action | Details |
|--------|---------|
| **Email blast** | "It's live" + direct store link + ask for honest review |
| **Product Hunt** | Launch Tue–Thu; prepare maker comment, 3 screenshots, 1-liner |
| **Indie Hackers** | "Launch" post + link to build story |
| **Hacker News** | "Show HN" only if dev angle is interesting |
| **Local communities** | Facebook groups, forums for your niche (first aid, outdoor, parents) |
| **Ask for reviews** | In-app prompt after user completes something useful (not on first open) |

### Launch Week Content Calendar

| Day | Post |
|-----|------|
| Mon | "Launching Thursday — here's the problem" |
| Tue | Demo video (60 sec) |
| Wed | Beta user quote / testimonial |
| Thu | **Launch day** — all channels |
| Fri | "First 24h stats + lessons" (builds trust) |
| Sat–Sun | Reply to every comment and review |

**Budget:** $0–$300 (optional: Product Hunt ship badge, small Apple Search Ads test)

**KPIs:** Installs, conversion rate (page → install), Day-1 retention, review count

## Phase 3: First 90 Days (Organic Growth Engine)

**Goal:** Steady installs without paid ads; learn what works.

### The Solo Founder "3-Channel Rule"

Pick **only 3** channels and go deep for 90 days:

**Recommended mix for utility/education apps:**

1. **ASO** — weekly keyword tweaks, A/B test screenshots every 4 weeks
2. **Content** — 1 short video or carousel per week (reuse app content)
3. **Community** — 30 min/day answering questions where your users already are

### Monthly Rhythm

| Week | Focus |
|------|-------|
| 1 | Analyze: which keywords, which posts drove installs |
| 2 | Double down on top channel |
| 3 | Ship one small feature users asked for → announce it |
| 4 | Refresh store listing (new screenshot, seasonal angle) |

### Low-Cost Paid Test (Optional, Month 2–3)

If you have **$300–$1,000** to test:

- **Apple Search Ads** (iOS) or **Google App Campaigns** (Android)
- Cap at **$10–$15/day** for 2 weeks
- Target your best ASO keywords only
- Stop if CPI > your expected revenue per user

**Budget:** $0–$1,000/month

**KPIs:** Organic vs paid installs, Day-7 retention (>15% is decent for utilities), store conversion rate (>25% is good)

## Phase 4: Months 4–12 (Compounding)

**Goal:** 1,000+ MAU or sustainable revenue without burning out.

### Growth Loops for Solo Founders

```
Content → Store listing → Install → In-app value → Review → Better ASO rank → More installs
```

| Tactic | Effort | Impact |
|--------|--------|--------|
| **App Store featuring request** | 2 hours once | High if accepted |
| **Partnerships** | Email 10 small orgs (scout groups, gyms, schools) | Medium |
| **Seasonal campaigns** | "Summer safety", "Holiday first aid kit" | Medium |
| **Referral in-app** | "Share with family" after user saves a guide | Medium |
| **Press / podcasts** | Pitch 5 niche podcasts (health, outdoors, parenting) | Medium |
| **Localization** | 1 new language = new keyword market | High over time |

### What to Avoid as a Solo Founder

- Running ads on 5 platforms at once
- Influencer campaigns under $5k (poor ROI for reference apps)
- Rebuilding for "virality" instead of retention
- Ignoring 1-star reviews (reply professionally, fix bugs fast)

**Budget:** $500–$3,000/year (tools, occasional ads, conference ticket)

**KPIs:** MAU, retention D30, revenue or "cost per engaged user", review average (aim 4.5+)


## 90-Day Action Plan

### Month 1

- [ ] Finalize store listing + landing page
- [ ] Post 3×/week on one social platform
- [ ] Join 2 communities; help, don't sell
- [ ] Beta with 30 users

### Month 2

- [ ] Launch on Product Hunt + email list
- [ ] Get 20 reviews in first 2 weeks
- [ ] Start 1 weekly short-form video
- [ ] Run $200 Apple/Google ad test

### Month 3

- [ ] A/B test screenshots
- [ ] Publish 2 SEO articles
- [ ] Pitch 5 podcasts or newsletters
- [ ] Ship 1 feature from user feedback + announce

## First Aid / Safety App Tips

Trust beats hype. Emphasize:

- **Credibility:** cite sources, disclaimers ("not a substitute for training"), clear author bio
- **Seasonal hooks:** summer injuries, travel, sports season
- **Partners:** outdoor retailers, hiking clubs, parenting blogs (affiliate or cross-promo, not paid influencers)
- **Offline:** QR code on a printable PDF "emergency card" → app download
- **App Store category:** Medical or Education — pick where competition vs. discoverability balance is best
